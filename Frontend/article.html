Asset-based audit workflows

Mar 8, 2025

The core logic of COD-DATA separates assets by Type to trigger specific audit checklists. A server gets different checks than a switch; an Exchange server has different requirements than a generic file server.

Each asset has an type and optional subType. For example: Server / Exchange or Switch / Layer 3. The system uses these to load the right checklist template.

Asset structure

Assets include: id, name, type, subType, ip, vulnerabilities array, and auditStatus (Pending, In-Progress, Complete). This structure lets you track both scan results and manual audit progress in one place.

Best practices

  • Define clear type/subType combinations for each asset category
  • Map each combination to a checklist template
  • Update auditStatus as you complete each section
  • Link vulnerability findings to the relevant asset

With this approach, consultants can perform consistent, thorough audits across diverse infrastructure while keeping each asset's context clear.
